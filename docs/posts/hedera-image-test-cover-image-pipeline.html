Introduction

This post tests the new end-to-end pipeline that generates a cover image for every blog using Gemini's Imagen 4 fast model.

How it works

  • One image per blog (cached by slug)
  • Cheapest Imagen model (~$0.02 per image)
  • Image is embedded in the HTML at the top of the article
  • Falls back gracefully if the API is unavailable

Key takeaways

  • Cost-controlled: never more than one image per post
  • Cached: re-publishing same slug uses existing image
  • Resilient: blog still publishes if image gen fails
